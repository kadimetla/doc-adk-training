# Module 35: Multimodal and Image Processing

## Lab 35: Building a Visual Product Catalog Analyzer

### Goal

In this lab, you will build a multi-agent system that can analyze a product image and generate a marketing description. This will teach you how to handle image inputs and orchestrate a workflow between a vision-capable agent and a text-generation agent.

### The Architecture

1.  **Vision Analyzer Agent:** An `LlmAgent` using a vision-capable model (`gemini-1.5-flash`) that takes an image and a text prompt, and outputs a structured analysis of the image.
2.  **Catalog Generator Agent:** A second `LlmAgent` that takes the structured analysis from the first agent and uses it to generate a polished, marketing-ready product description.

### Step 1: Create the Agent Project

1.  **Create the agent project:**
    ```shell
    adk create visual-catalog
    ```
    When prompted, choose the **Programmatic (Python script)** option.

2.  **Navigate into the new directory:**
    ```shell
    cd visual-catalog
    ```

### Step 2: Implement the Multimodal Agents

**Exercise:** Open `agent.py` and replace its contents with the full solution from the `lab-solution.md`.

Your task is to study this code and understand its key components:

1.  **`ProductCatalogAnalyzer` Class:** This class encapsulates the entire workflow.
2.  **`vision_agent`:**
    *   Notice the `model` is set to `gemini-1.5-flash`, which can process images.
    *   Its `instruction` is tailored to analyzing product images.
3.  **`catalog_agent`:**
    *   This is a standard text-based agent. Its instruction tells it how to use the analysis from the first agent to write a description.
4.  **`analyze_product` Method:**
    *   **Image Loading:** See how `load_image_from_file` is used to read an image from disk and create a `types.Part` object.
    *   **Multimodal Query:** The `analysis_query` is a **list** containing both a `text` part and the `image` part. This is the key to sending multimodal input.
    *   **Orchestration:** The method first `await`s the result from the `vision_agent`, then uses that result to construct a new query for the `catalog_agent`.

### Step 3: Run and Test the Vision Agent

1.  **Set up your `.env` file for Vertex AI.** Vision models require a Vertex AI configuration.

2.  **Run the agent script directly:**
    The solution file includes a `main` block that will create dummy images and run the analysis pipeline.

    ```shell
    uv run python agent.py
    ```

3.  **Analyze the Output:**
    The script will run the full analysis for three different products. For each one, you will see:
    1.  The detailed, structured analysis generated by the `vision_agent` after "looking" at the image.
    2.  The final, marketing-ready catalog entry generated by the `catalog_agent` based on that analysis.
    3.  A final summary of all the products that were processed.

### Lab Summary

You have successfully built a multi-agent system that can see and understand images!

You have learned to:
*   Configure an agent to use a vision-capable model like `gemini-1.5-flash`.
*   Create multimodal prompts by combining text and image `types.Part` objects in a list.
*   Build a pipeline where one agent's visual analysis becomes the input for another agent's text generation task.
*   Understand the fundamentals of building vision-enabled AI agents.
